# Change Constipation

If you were under the impression this blog would make meaningful contributions to the world… you were right!

"Model capability overhang", or Change constipation as I am now coining it, is very real.

The human bottleneck of understanding LLM's is both faster than anything that's come before it, and yet just as biased as past technological shifts.

I really liked Yasin's recent blog on this topic, reiterating the nuance and context it takes to be using these tools to the best of their ability.

> The skill isn't writing code anymore. The skill is knowing what to build and how it should work. The code is just the output.

I go back and forth wondering if it's actually worth being at the bleeding edge, but for now it seems beneficial. However I have noticed one slippery slope I've fallen victim to.

Claude Code's creator Boris Cherny has talked at length about how they built "Claude Code" for the model 6 months out. But this requires discipline. I built Magpai for the models 6 months out, and then I kept building it, and building it. The 6 months kept passing, and I kept shifting the goal posts… This resulted in a large pivot from GenAI Browser Nodes -> AI productivity operating system. In hindsight? I should have shipped the nodes, and held out a few more months (As indeed Weavy.ai and flora.ai did).

Don't get constipated with the possibility of tomorrow, or the legacy of the past.
